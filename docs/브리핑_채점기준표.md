# 경제번역기 브리핑 채점 기준표 (Rubric)

**현재 버전: v2.7** · 최초 작성 2026-07-05 (채점기 세션1) · v2.2 개정 2026-07-12 (점수 첫 주 리뷰) · v2.3 개정 2026-07-20 (오판 3·4호 리뷰) · **v2.4~v2.7 개정 2026-07-20 (halo 차단·톤 폐기·사실 지목 강제·모델 상향)**
**목적**: 매일 아침 자동 생성되는 경제 브리핑의 품질을, 감(感)이 아니라 정해진 항목으로 점수화한다. 이 문서가 나중에 자동 채점기(LLM-as-judge)의 채점 지시문 원본이 된다.

---

## 0. 배경 — 왜 채점기보다 기준표가 먼저인가

채점 자동화를 먼저 붙이면 "AI가 매긴 점수"가 나와도 **뭘 재는 점수인지 아무도 모른다.**
사람이 먼저 "무엇이 좋은 브리핑인가"를 정의(=이 기준표)하고 → 그걸 AI에게 채점시키는 순서가 정석이다.
이 기준표는 그 정의 문서이고, 자동 채점 구현(P1)은 이 문서를 코드로 옮기는 일이다.

## 2. 점수 항목 4개 (각 0~2점, **총 8점** — v2.4에서 톤 제외)

> **v2.4 채점 방식 변경 — 항목별 개별 호출.** 옛 방식은 한 번의 AI 호출로 전 항목을 채점해
> 한 항목의 강한 감점이 나머지를 오염시켰다(halo). 2026-07-09 사고 때 "항목별 독립 채점"
> 지시를 프롬프트에 넣었지만, 7/20 검증에서 사실 0점이 나오자 이해도 2→0·톤 2→1로 동반
> 하락 — **지시만으로는 안 지켜짐이 실증**됐다. → 항목마다 별도 호출로 분리해, 심사위원이
> 다른 항목의 판단을 **구조적으로 볼 수 없게** 했다.

척도: **2=충족 / 1=보통 / 0=미달**

### ① 이해도 — 경제 초보자가 바로 이해되는가 (서비스의 핵심 약속)
- **적용 부위**: 전체 요약 · 개별 해설 '쉽게 설명' · 오늘의 용어
- **채점 방법**: AI 채점 (브리핑 글만 봐도 판단 가능)
- 2점: 중학생도 이해. 어려운 말은 다 풀어썼고 예시로 감이 온다
- 1점: 대체로 이해되나 일부 용어가 설명 없이 나온다
- 0점: 전문용어를 그대로 써서 무슨 말인지 어렵다

### ② 사실 정확성 — 숫자·인과·출처가 맞는가, 지어낸 정보는 없는가 (신뢰 직결)
- **적용 부위**: 숫자 방향 · 원인→결과 · 헤드라인 · 개별 해설
- **채점 방법**: AI 채점 **+ 원문 기사 첨부 필수** (아래 3.① 참조)
- ⚠️ **P1 구현 메모(2026-07-07)**: 기사 본문은 DB에 저장하지 않아(제목+주소만) 실제 첨부 가능한 대조 근거 = **그날 기사 제목 풀 + 실측 지표 값**까지다. 제목·지표로 확인 불가한 서술은 감점하지 않고 채점 근거(reason)에 "확인 불가"로만 남긴다. 이 한계는 점수 행 inputs.factualBasis에도 매번 기록된다. 본문 대조가 필요해지면 그때 원문 수집을 별도 설계한다(지금은 비용>효용으로 보류).
- 2점: 원문과 대조해 숫자·오르내림 방향·인과가 모두 정확
- 1점: 큰 오류는 없으나 사소하게 부정확하거나 모호한 표현이 있다
- 0점: 숫자가 틀림 · 오르내림 방향이 반대 · 원문에 없는 내용이 있다
- ⚠️ **원문 없이 채점하면 점수를 매기지 말고 '판정 불가'로 기록.** 원문 대조 없는 사실 점수는 인상 평가일 뿐 무효다.
- ⚠️ **시제 검사 (v2.3)**: 후보 기사가 "오늘 ○○ 여부를 결정한다/전망"인데 브리핑이 "○○했어요"로 **단정**했으면 **사실 0점 + fabrication 검토**. 실측 지표가 아직 옛 값인 것은 "브리핑이 맞다"는 근거가 아니다(발표 전이라 안 바뀐 것). — 계기: 2026-07-16 금통위 인상일에 "동결했어요"라 쓴 브리핑이 **총점 10점 만점**을 받음(오판 3호).
- ⚠️ **내부 일관성 검사 (v2.3)**: 헤드라인·요약·shareCard와 TOP3 해설이 서로 반대로 말하면 둘 중 하나는 틀린 것이므로 **사실 1점 이하**. 채점기 프롬프트에 TOP3 해설 요지를 함께 넣어 대조 가능하게 함(옛 버전은 제목만 줘서 모순을 볼 수 없었다).

### ③ 선정 판단 — 그날 가장 중요한 이슈를 골랐는가 (편집력)
- **적용 부위**: 헤드라인 · TOP3
- **채점 방법**: AI 채점 **+ 그날 후보 기사 풀 첨부 필수** (아래 3.① 참조)
- 판단 기준(생성 프롬프트와 동일): 파급(영향받는 사람 수)·체감(내 지갑에 닿는 정도)·사건성(오늘 새로 터졌나)·규모(변화 크기)를 종합
- 2점: 후보 중 가장 중요한 이슈를 1등으로 올렸다
- 1점: 무난하나 더 중요한 걸 놓쳤다
- 0점: 사소한 걸 1등으로 올림 · 시황/해외 단독 뉴스를 올림
- ⚠️ **후보 풀 없이 채점하면 '판정 불가'로 기록.** 최종 브리핑만 보면 "버린 것 중 더 중요한 게 있었는지" 알 수 없다.
- ⚠️ **근거 지목 강제 (v2.2 → v2.3 개정)**: 감점하려면 후보 풀에서 놓친 기사 제목을 **그대로** 지목해야 하고, 코드가 검증한다. 단순 시황·해외 단독은 "놓친 기사"로 지목 금지.
  - **v2.3 ⓐ 지목 실패 = 2점 → `판정 불가(null)`**. 옛 방식은 판정 불가를 **만점으로 기록**해 점수를 부풀렸다(7/12·7/19 실측).
  - **v2.3 ⓑ 이미 TOP3에 선정된 기사는 지목 불가** — 코드가 TOP3 제목과 대조해 무효 처리(`[지목 무효 → 판정 불가]`). 옛 가드는 "후보 풀에 있는가"만 봐서, TOP3는 후보 풀 출신이라 **항상 통과**했다(7/20 실측, 오판 4호).
- ※ "가장 중요한 뉴스"는 본질적으로 주관이 섞여 점수가 흔들릴 수 있음을 감안한다.


### ④ 다양성 — 뉴스가 겹치지 않고 골고루인가
> **v2.4 검토 결과: 코드 검사로 옮기지 못함(보류).** 단어 겹침(titleSimilarity)으로는 "같은 주제·다른 표현"을 못 잡는다 —
> 실제 중복이었던 7/19 TOP3 ②③(둘 다 증권사 2분기 실적)의 겹침 계수가 **0.111**로, 정상적으로 다양했던 날들(0.000)과
> 구분되지 않는다. 임계값을 0.111까지 낮추면 단어 하나만 공유해도 겹침으로 잡힌다.
> 의미 유사도(embedding)는 그 쌍을 **0.526으로 최고값**에 올려 가능성을 보였으나, 라벨된 사례가 1건뿐이고
> 0.39~0.48 구간과 간격이 얇아 **지금 임계값을 정하면 근거 없는 추측**이 된다(규칙 13).
> → **판정에는 쓰지 않고 매일 `inputs.topicSimilarity`에 값만 기록**한다. 2~3주 쌓인 뒤 사람이 임계값을 정한다.
> (측정을 먼저 붙이고 자동 판정은 나중에.)
- **적용 부위**: TOP3 서로 다른 사건·주제 · 분야 편중 · 시황/해외 단독 배제
- **채점 방법**: AI 채점
- 2점: TOP3 3개가 다 다른 주제, 분야가 고르고, 시황·해외 단독이 없다
- 1점: 대체로 다양하나 주제 하나가 겹친다
- 0점: 같은 사건이 중복되거나, 빈자리를 시황·해외로 채웠다
- ⚠️ **근거 지목 강제 (v2.2 → v2.3 개정)**: 감점하려면 TOP3 중 어느 두 개(번호)가 어떤 주제로 겹치는지 지목해야 하고, 코드가 번호 유효성을 검증한다. **특정 못 하면 v2.3부터 `판정 불가(null)`**(옛 방식=2점 보정). — 계기: 첫 주 복붙형 1점 고정 → v2.2로 개선했으나, 이번엔 판정 불가가 만점으로 새는 반대 문제가 확인됨(7/19 겹침을 못 보고 2점, 7/20 같은 유형을 지목 성공 = 일관성 없음).

### ⑤ 톤 — **v2.4에서 점수 항목 폐기**
> 7/13~7/20 8일 내내 2점 고정 = 변별력 0인데 총점만 2점씩 올려 "점수가 높다"는 착시를 만들었다.

**다음 중 하나면 그날 채점은 🚨 경보:**

- 하드 형식 탈락

경보 채널: **점수 행 issue_note에 `🚨 경보: ...` 박제(→ 점수 시트에 그대로 표시) + 보조로 notifyFailure 메일·서버 로그.** 사용자는 메일을 잘 안 보므로 시트·주간 리뷰가 1차 확인 지점이다.

## 4. 반드시 지킬 3원칙

1. **입력 첨부 필수** — 사실(②)·선정(③) 채점에는 원문 기사 / 그날 후보 풀을 함께 넣는다. 없으면 그 항목은 '판정 불가'. (ground truth 없는 채점은 무효)
2. **버전 + 갱신 트리거** — 이 기준표는 버전을 달고, 아래 트리거가 걸리면 갱신하고 이력에 남긴다.
3. **채점은 발행과 분리(비동기)** — 채점 AI 호출이 실패해도 브리핑 발행은 절대 막히면 안 된다. 채점은 발행 경로 밖에서 별도로 돈다.

## 6. 버전 이력 & 갱신 트리거

**갱신 트리거** (하나라도 발생하면 기준표 재검토 → 버전 상향 → 이력 기록):
- 브리핑 생성 프롬프트를 크게 바꿨을 때
- 월간 결산에서 '기준표가 재는 것'과 '실제 중요한 것'의 불일치를 발견했을 때
- P2(경보선 설정) 진행 시

**이력**:
- **v2.7 (2026-07-20)** — 지표 함정 차단. 7/16이 gpt-4o에서도 사실 2점을 받은 원인 = 그날 지표 '기준금리 2.50% (— 동결)'을 심사위원이 "동결 결정 확인"으로 읽음. 실제로는 '직전 거래일 대비 무변화'(발표 전이라 안 바뀐 것). → **후보 풀에 "오늘 결정" 기사가 있는 날엔 지표를 브리핑이 맞다는 근거로 쓸 수 없음**을 명문화 + 시제 검사를 숫자 대조보다 **선행 단계**로. ✅검증: 7/16 사실 0점·🚨경보 정상 발동, 7/17(정상일)은 시제 위반 아님으로 올바르게 통과.
- **v2.6 (2026-07-20)** — **사실 항목만 모델 상향(gpt-4o-mini → gpt-4o)**. 근거=13일치 실측에서 mini가 근거를 인용해놓고 자기모순 결론을 냄("기준금리 2.75% 인상"을 인용하며 "인상 서술 불일치", "하락한 것이 맞음에도 불구하고 일치하지 않음"). 코드 가드는 *인용이 실재하는가*까지만 검증할 수 있고 *추론이 타당한가*는 못 본다 → 판단 난이도가 가장 높은 항목만 모델을 올림(하루 1회라 비용 영향 미미). 상향 후 7/12·7/15·7/17·7/20의 헛 감점이 정상화됨.
- **v2.5 (2026-07-20)** — **사실 감점에도 근거 지목 강제**(선정·다양성에 쓴 원칙 확대 적용). 계기=v2.4 전체 재채점에서 13일 중 9일이 사실 0점으로 튐. 심사위원이 시제 규칙을 "지표로 확인 안 되면 0점"으로 뭉개 적용(이미 일어난 인상을 '발표 전 단정'이라 하고, 전망 서술·확인 불가까지 0점 — 전부 프롬프트가 금지한 것). → ①시제 검사를 **3조건 AND**로 좁힘(후보 풀에 미결정 기사 존재 + 완료형 단정 + 지표 미반영) ②감점 시 claim/evidence 지목 필수, 코드가 대조 자료와 토큰 대조로 검증 ③검증 실패 시 감점이 아니라 **판정 불가**.
- **v2.4 (2026-07-20)** — 같은 날 v2.3 검증 중 드러난 halo 재발 대응. ①**항목별 개별 호출로 분리** — 프롬프트 지시로는 전염이 안 막혔음이 실증돼 구조로 차단 ②**톤 점수 항목 폐기**(8일 내내 2점, 변별력 0) → 존댓말 일관성만 형식 소프트 검사로 이동. 만점 10점 → **8점** ③다양성 코드 이관은 **데이터 부족으로 보류**, 대신 의미 유사도를 매일 관측 기록(`inputs.topicSimilarity`) ④다양성 감점 시 "금융·경제" 같은 **넓은 범주로 묶는 것 금지** 명문화(같은 사건·주체·통계여야 겹침) ⑤경보선 재조정(만점 절반 이하 / 판정 불가 2항목 이상).
- **v2.3 (2026-07-20)** — 오판 3·4호 리뷰 개정(사고기록=`전직로드맵/경제번역기_채점기_오판사고_2026-07-20.md`). ①**지목 가드 양방향 수정** — 지목 실패/무효 시 2점이 아니라 **판정 불가(null)**, 이미 TOP3에 있는 기사는 지목 불가 ②**시제 검사** 신설(발표 전 사안을 단정하면 사실 0점) ③**내부 일관성 검사** 신설 + 채점기에 TOP3 해설 요지 제공 ④**총점을 판정된 항목만 합산**하고 분모(`inputs.scoreDenominator`)를 함께 기록 — 옛 방식은 하나만 판정 불가여도 총점이 통째로 사라졌다 ⑤경보선을 절대점수→**만점 대비 비율**로, 판정 불가 3항목 이상 경보 신설. 계기=7/16 "동결" 오보가 10점 만점, 7/20 TOP3 소속 기사를 "놓쳤다" 지목이 가드 통과.
- **v2.2 (2026-07-12)** — 점수 첫 주 리뷰(P2 트리거) 개정. ①선정·다양성에 **근거 지목 강제** — 감점 시 놓친 기사 제목/겹치는 TOP3 번호를 지목해야 하고 코드가 검증, 실패 시 감점 무효(2점 보정). 계기=5일 내내 헤지형 1점 고정 + 심사위원이 시황을 "더 중요"라 한 자기 기준 위반 3회 ②형식 검사 **하드/소프트 2단계** — 탈락은 하드만, 소프트는 경고 기록 ③헤드라인 첫 줄 18→**20자** 완화(소프트, 생성기 무수정) ④**P2 경보선 확정**(실격·사실 0점·하드 탈락·총점 ≤5 → 🚨, §3.③).
- **v2.1 (2026-07-05)** — 최초 확정. v1(이해도·사실·형식·톤·다양성 동일배점 5점)에서 다음을 수정: ①형식을 점수→코드검사로 이동 ②투자조언을 톤→안전선(실격)으로 이동 ③빠졌던 '선정 판단'을 점수 항목에 추가 ④사실·선정에 입력 첨부 필수 명시 ⑤척도 5점→0/1/2 + 매일 문제 한 줄 메모 병행.
