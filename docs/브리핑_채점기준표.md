# 경제번역기 브리핑 채점 기준표 (Rubric)

**현재 버전: v2.2** · 최초 작성 2026-07-05 (채점기 세션1) · v2.2 개정 2026-07-12 (점수 첫 주 리뷰)
**목적**: 매일 아침 자동 생성되는 경제 브리핑의 품질을, 감(感)이 아니라 정해진 항목으로 점수화한다. 이 문서가 나중에 자동 채점기(LLM-as-judge)의 채점 지시문 원본이 된다.

---

## 0. 배경 — 왜 채점기보다 기준표가 먼저인가

채점 자동화를 먼저 붙이면 "AI가 매긴 점수"가 나와도 **뭘 재는 점수인지 아무도 모른다.**
사람이 먼저 "무엇이 좋은 브리핑인가"를 정의(=이 기준표)하고 → 그걸 AI에게 채점시키는 순서가 정석이다.
이 기준표는 그 정의 문서이고, 자동 채점 구현(P1)은 이 문서를 코드로 옮기는 일이다.

---

## 1. 채점 대상 — 브리핑이 실제로 만들어내는 요소

(generateBriefing.ts / PROMPT-ENGINEERING.md 원본 기준)

1. 헤드라인 2줄 (첫 줄 18자 이내, 그날 가장 큰 이슈)
2. 전체 요약 3~5문단 (초보자 언어)
3. 오늘의 한 줄 (shareCard, 20~40자, 카톡 공유용)
4. 오늘의 용어 (dailyTerm)
5. TOP3 뉴스 (서로 다른 사건 3개)
6. 분야별 건강진단 (healthCheck: 물가·소비·수출·고용·부동산·금융 등)
7. 핵심 흐름 (connections 3~5개)
8. 개별 뉴스 해설 5단 (무슨 기사 / 쉽게 설명 / 왜 중요 / 한국 영향 / 핵심 포인트)

## 2. 점수 항목 5개 (각 0~2점, 총 10점)

척도: **2=충족 / 1=보통 / 0=미달**

### ① 이해도 — 경제 초보자가 바로 이해되는가 (서비스의 핵심 약속)
- **적용 부위**: 전체 요약 · 개별 해설 '쉽게 설명' · 오늘의 용어
- **채점 방법**: AI 채점 (브리핑 글만 봐도 판단 가능)
- 2점: 중학생도 이해. 어려운 말은 다 풀어썼고 예시로 감이 온다
- 1점: 대체로 이해되나 일부 용어가 설명 없이 나온다
- 0점: 전문용어를 그대로 써서 무슨 말인지 어렵다

### ② 사실 정확성 — 숫자·인과·출처가 맞는가, 지어낸 정보는 없는가 (신뢰 직결)
- **적용 부위**: 숫자 방향 · 원인→결과 · 헤드라인 · 개별 해설
- **채점 방법**: AI 채점 **+ 원문 기사 첨부 필수** (아래 3.① 참조)
- ⚠️ **P1 구현 메모(2026-07-07)**: 기사 본문은 DB에 저장하지 않아(제목+주소만) 실제 첨부 가능한 대조 근거 = **그날 기사 제목 풀 + 실측 지표 값**까지다. 제목·지표로 확인 불가한 서술은 감점하지 않고 채점 근거(reason)에 "확인 불가"로만 남긴다. 이 한계는 점수 행 inputs.factualBasis에도 매번 기록된다. 본문 대조가 필요해지면 그때 원문 수집을 별도 설계한다(지금은 비용>효용으로 보류).
- 2점: 원문과 대조해 숫자·오르내림 방향·인과가 모두 정확
- 1점: 큰 오류는 없으나 사소하게 부정확하거나 모호한 표현이 있다
- 0점: 숫자가 틀림 · 오르내림 방향이 반대 · 원문에 없는 내용이 있다
- ⚠️ **원문 없이 채점하면 점수를 매기지 말고 '판정 불가'로 기록.** 원문 대조 없는 사실 점수는 인상 평가일 뿐 무효다.

### ③ 선정 판단 — 그날 가장 중요한 이슈를 골랐는가 (편집력)
- **적용 부위**: 헤드라인 · TOP3
- **채점 방법**: AI 채점 **+ 그날 후보 기사 풀 첨부 필수** (아래 3.① 참조)
- 판단 기준(생성 프롬프트와 동일): 파급(영향받는 사람 수)·체감(내 지갑에 닿는 정도)·사건성(오늘 새로 터졌나)·규모(변화 크기)를 종합
- 2점: 후보 중 가장 중요한 이슈를 1등으로 올렸다
- 1점: 무난하나 더 중요한 걸 놓쳤다
- 0점: 사소한 걸 1등으로 올림 · 시황/해외 단독 뉴스를 올림
- ⚠️ **후보 풀 없이 채점하면 '판정 불가'로 기록.** 최종 브리핑만 보면 "버린 것 중 더 중요한 게 있었는지" 알 수 없다.
- ⚠️ **근거 지목 강제 (v2.2)**: 감점하려면 후보 풀에서 놓친 기사 제목을 **그대로** 지목해야 하고, 코드가 후보 풀과 대조해 검증한다. 지목이 없거나 대조 실패면 감점 무효(2점 보정, reason에 `[지목 실패 → 감점 무효]` 표기). 단순 시황·해외 단독은 "놓친 기사"로 지목 금지(심사위원 자기 기준 위반 3회 사례 차단). — 계기: 첫 주 5일 내내 "놓쳤을 여지" 헤지형 1점 고정.
- ※ "가장 중요한 뉴스"는 본질적으로 주관이 섞여 점수가 흔들릴 수 있음을 감안한다.

### ④ 다양성 — 뉴스가 겹치지 않고 골고루인가
- **적용 부위**: TOP3 서로 다른 사건·주제 · 분야 편중 · 시황/해외 단독 배제
- **채점 방법**: AI 채점
- 2점: TOP3 3개가 다 다른 주제, 분야가 고르고, 시황·해외 단독이 없다
- 1점: 대체로 다양하나 주제 하나가 겹친다
- 0점: 같은 사건이 중복되거나, 빈자리를 시황·해외로 채웠다
- ⚠️ **근거 지목 강제 (v2.2)**: 감점하려면 TOP3 중 어느 두 개(번호)가 어떤 주제로 겹치는지 지목해야 하고, 코드가 번호 유효성을 검증한다. 특정 못 하면 감점 무효(2점 보정). — 계기: 첫 주 5일 내내 "일부 주제 겹침" 복붙형 1점 고정.

### ⑤ 톤 — 문체가 일관된가
- **적용 부위**: 존댓말 · 1인칭 '우리' · 짧고 명확한 문장 · 이모지 적정
- **채점 방법**: AI 채점
- 2점: 따뜻하고 일관되며 이모지가 적정하다
- 1점: 대체로 일관하나 일부 문장이 딱딱하거나 이모지가 과하다
- 0점: 반말이 섞이거나 문체가 들쭉날쭉하고 이모지를 남발한다
- ※ '투자조언 금지'는 톤이 아니라 안전선(4.② 실격)으로 분리했다.

## 3. 점수 밖 항목 2개 (점수와 별개로 판정)

### ① 형식 — 코드로 기계적 검사 (AI에게 맡기지 않음)
글자·문단·개수 세기는 AI가 자주 틀리므로 **코드로 검사**한다.

**v2.2부터 하드/소프트 2단계** (계기: 7/11 헤드라인 1자 초과가 빈 제목 사고와 똑같이 그날 전체 형식❌로 찍힘 — 가드레일의 심각도 분류와 같은 원리):
- **하드** = 독자 화면이 깨지거나 내용 누락 → 하나라도 실패면 그날 **형식 탈락**(format_pass=false)
- **소프트** = 글자수·문단수 소폭 이탈 → **경고로 기록만** (issue_note에 ⚠️ 표기, 탈락 아님)

| 검사 | 등급 |
|---|---|
| 헤드라인 2줄 구조 | 하드 |
| 헤드라인 첫 줄 **20자** 이내 *(v2.2: 18→20자 완화 — 7/11 19자 사례, 생성기는 무수정)* | 소프트 |
| shareCard 20~40자 | 소프트 |
| summary 3~5문단 | 소프트 |
| TOP3 정확히 3개 | 하드 |
| TOP3 제목 빈칸 없음 *(2026-07-08 추가 — UUID 한 글자 누락 사고 계기)* | 하드 |
| healthCheck 분야 최소 4개 채워짐 | 하드 |
| JSON 필드 안 깨짐 · 빈 칸 없음 | 하드 |

### ② 안전선 — 하나라도 걸리면 그날 브리핑 '실격' (점수와 무관)
- 투자 조언/권유 문구가 있다
- 원문에 없는 사실을 지어냈다(fabrication) — ※ 자동 감지는 원문 대조가 필요하고 한계가 있음. 감지되면 실격 처리
- → 점수가 아무리 높아도 실격이면 그날 브리핑은 fail로 별도 표기한다.

### ③ 경보선 — P2 (v2.2 확정, 2026-07-12 첫 주 리뷰)
첫 주 5일 점수(6·7·7·7·6)가 안정적이라 기준선을 확정. **신뢰하는 신호만 조건으로 쓴다** — 선정·다양성은 신뢰 보류 중(2026-07-11 의사결정기록)이라 경보 조건에서 제외.

**다음 중 하나면 그날 채점은 🚨 경보:**
- 실격 (투자조언·fabrication)
- 사실 0점
- 하드 형식 탈락
- 총점 5점 이하 (관측 바닥 6점 아래 — 평소엔 안 울리고 진짜 이상만)

경보 채널: **점수 행 issue_note에 `🚨 경보: ...` 박제(→ 점수 시트에 그대로 표시) + 보조로 notifyFailure 메일·서버 로그.** 사용자는 메일을 잘 안 보므로 시트·주간 리뷰가 1차 확인 지점이다.

## 4. 반드시 지킬 3원칙

1. **입력 첨부 필수** — 사실(②)·선정(③) 채점에는 원문 기사 / 그날 후보 풀을 함께 넣는다. 없으면 그 항목은 '판정 불가'. (ground truth 없는 채점은 무효)
2. **버전 + 갱신 트리거** — 이 기준표는 버전을 달고, 아래 트리거가 걸리면 갱신하고 이력에 남긴다.
3. **채점은 발행과 분리(비동기)** — 채점 AI 호출이 실패해도 브리핑 발행은 절대 막히면 안 된다. 채점은 발행 경로 밖에서 별도로 돈다.

## 6. 버전 이력 & 갱신 트리거

**갱신 트리거** (하나라도 발생하면 기준표 재검토 → 버전 상향 → 이력 기록):
- 브리핑 생성 프롬프트를 크게 바꿨을 때
- 월간 결산에서 '기준표가 재는 것'과 '실제 중요한 것'의 불일치를 발견했을 때
- P2(경보선 설정) 진행 시

**이력**:
- **v2.2 (2026-07-12)** — 점수 첫 주 리뷰(P2 트리거) 개정. ①선정·다양성에 **근거 지목 강제** — 감점 시 놓친 기사 제목/겹치는 TOP3 번호를 지목해야 하고 코드가 검증, 실패 시 감점 무효(2점 보정). 계기=5일 내내 헤지형 1점 고정 + 심사위원이 시황을 "더 중요"라 한 자기 기준 위반 3회 ②형식 검사 **하드/소프트 2단계** — 탈락은 하드만, 소프트는 경고 기록 ③헤드라인 첫 줄 18→**20자** 완화(소프트, 생성기 무수정) ④**P2 경보선 확정**(실격·사실 0점·하드 탈락·총점 ≤5 → 🚨, §3.③).
- **v2.1 (2026-07-05)** — 최초 확정. v1(이해도·사실·형식·톤·다양성 동일배점 5점)에서 다음을 수정: ①형식을 점수→코드검사로 이동 ②투자조언을 톤→안전선(실격)으로 이동 ③빠졌던 '선정 판단'을 점수 항목에 추가 ④사실·선정에 입력 첨부 필수 명시 ⑤척도 5점→0/1/2 + 매일 문제 한 줄 메모 병행.
