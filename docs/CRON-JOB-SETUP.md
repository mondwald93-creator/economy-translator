# cron-job.org 설정 가이드 — 아침 브리핑 9시 정각 실행

## 왜 필요한가 (2026-06-11 진단 결과)

1. **정각 문제**: Vercel 무료 요금제의 예약 실행은 "9시 정각"이 아니라 "9시~10시 사이 아무 때나" 실행됨 (실측: 9:16~9:54)
2. **타임아웃 문제 (진짜 원인)**: 기존 `/api/cron`은 "뉴스 수집 + AI 해설 생성 + 메일 발송"을 한 번에 하는데, 서버의 작업당 시간 제한이 5분이라 수집이 오래 걸린 날은 해설 생성 전에 잘림. 6/11 실제 발생: 9:54 뉴스 623건 수집 성공 → 브리핑 생성 전 시간 초과로 사망.
   - 실측: 수집 1.5~4분 + 해설 생성 약 3분 = 합치면 5분 초과 위험. **따로 하면 각각 5분 안에 여유 있게 끝남.**

→ 해결: cron-job.org(무료, 분 단위 정각 보장)에서 **두 단계로 나눠서** 호출.
→ 기존 Vercel 예약은 지우지 않고 안전망으로 유지 (이중 실행돼도 같은 날짜 덮어쓰기라 무해).

## 설정 절차 (1회, 약 10분)

1. **가입**: https://cron-job.org → SIGN UP (무료, 이메일 인증)
2. **크론잡 2개**를 아래처럼 만든다 (CREATE CRONJOB 두 번):

### 작업 1 — 아침 뉴스 수집 (9:00 정각)

| 항목 | 값 |
|------|-----|
| Title | 경제번역기 1-뉴스수집 |
| URL | `https://economy-translator.vercel.app/api/cron-news` |
| Schedule | Every day at **9:00** |
| Timezone | **Asia/Seoul** (기본값이 유럽 시간대이므로 반드시 변경) |

### 작업 2 — 브리핑 생성 (9:07)

| 항목 | 값 |
|------|-----|
| Title | 경제번역기 2-브리핑생성 |
| URL | `https://economy-translator.vercel.app/api/cron-briefing` |
| Schedule | Every day at **9:07** (수집이 끝난 뒤 시작되도록 7분 간격) |
| Timezone | **Asia/Seoul** |

3. **두 작업 모두** HEADERS 탭(또는 Advanced)에서 헤더 1개 추가:
   - Key: `Authorization`
   - Value: `Bearer <비밀암호>` — 비밀암호 값은 이 프로젝트 폴더의 `.env.local` 파일 안 `CRON_SECRET` 항목에 있음. "Bearer" 뒤에 한 칸 띄고 붙여넣기.
   - (이 암호가 없으면 우리 사이트가 "허가받지 않은 호출"로 거절함 — 2026-06-11 실서버 통과 확인 완료)
4. 저장 (CREATE)

→ 결과: 매일 **9시 12분 전후**에 그날 개장 데이터 기준 브리핑 게시.

## ⚠️ 꼭 알아둘 것: "시간 초과(timeout)" 표시는 실패가 아님

- cron-job.org는 **30초까지만** 응답을 기다림. 우리 작업은 각각 2~4분 걸림.
- 따라서 cron-job.org 실행 기록에는 매일 "timeout"이 뜨지만, **우리 서버는 연결이 끊겨도 작업을 끝까지 완료함.**
- 성공 여부는 cron-job.org 기록이 아니라 **사이트에 오늘 브리핑이 떴는지**로 판단.
- 설정에서 **알림(Notifications)을 꺼둘 것** — 안 끄면 매일 "실패" 메일이 옴.

## 설정 후 확인 방법

- 다음 날 아침 9시 12분쯤 https://economy-translator.vercel.app 접속 → 오늘 날짜 브리핑 확인
- 또는 두 작업을 순서대로(1번 → 5분 뒤 2번) TEST RUN 버튼으로 즉시 실행해보고 사이트 확인
